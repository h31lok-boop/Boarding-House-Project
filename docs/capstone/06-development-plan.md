# Step-by-Step Development Plan

## Phase 1: Planning and Foundations
- Finalize scope and role permissions.
- Define ERD and migration map.
- Standardize folder structure.

## Phase 2: Core Back-End Modules
- Auth + role gates (admin/owner/tenant).
- Boarding house CRUD with approval flow.
- Room CRUD with pricing and availability fields.
- Tenant and payment record modules.

## Phase 3: User Discovery Features
- Browse and filter by price, location, amenities, availability.
- House details page with map pin and amenities.
- Favorites module.

## Phase 4: Engagement Features
- Inquiry module.
- Reservation module with status lifecycle.
- Reviews and rating aggregation.

## Phase 5: Comparison + Geotagging Experience
- Multi-select compare page.
- Distance computation and table display.
- Map-centric browsing UI.

## Phase 6: Dashboards and Reporting
- Admin metrics and approvals.
- Owner listing, room, tenant, payment metrics.
- User dashboard with favorites, reservations, inquiries.

## Phase 7: QA and Capstone Readiness
- Seed realistic demo data.
- Add feature tests for critical flows.
- Polish UI and mobile responsiveness.
- Prepare documentation and demo script.
