# Boarding House Management and Comparison System with Geotagging

## Capstone Vision
A role-based web platform where:
- `Admin` governs users and listings.
- `Manager/Owner` manages properties, rooms, tenants, and payments.
- `User/Tenant` discovers, compares, favorites, and inquires/reserves boarding houses.

## Primary Goals
- Clean and responsive web UI.
- Geotagged map browsing.
- Multi-house comparison engine.
- Production-style architecture while staying beginner-friendly.

## Core Roles
- `admin`
- `owner` (manager)
- `tenant` (end user)

## Deliverables In This Phase
- Tech stack recommendation.
- Folder structure plan.
- Relational schema and SQL.
- ERD narrative.
- Full system architecture.
- Step-by-step development plan.
